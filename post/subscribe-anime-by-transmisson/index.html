通过Transmisson订阅rss追番

2024/Jul/05
4 minute read
0 visits
0 comments

网上的大部分教程都是qb的, 本篇教程是基于transmission的.

前面尝试过基于flexget的订阅, 自动下载没问题, 但是为了方便刮削需要重命名, 破坏了tr的种子分享, 而且手动维护flexget的config配置文件也很麻烦

接触 autobangumi 这个项目挺早的了, 作者最开始在nga宣传就看到了. 但是因为没有nas, 路由器跑不起来qbweb, 所以就放弃了

后面正好想学python, 看到pr里有tr的提交, 但没合并到main分支, 就拿有tr的分支开始动手修改.

目前自己用起来大体没什么问题.

==后面的步骤全程需要科学上网==

transmission 的安装与设置

以openwrt为例:

  1. 到软件里安装 luci-transmission 带zh-cn版本

  2. 进行相关设置:

    1. 配置文件目录改为非临时目录, 否则重启丢失下载进度
    2. 用户权限和组权限改为root
    3. 设置下载目录为实际目录
    4. 开启rpc功能, 勾选rpc授权验证, 设置用户名和密码
    5. 设置分享率限制, 例如为1
    6. 重启

  3. 自定义web功能:

    1. github 搜索 TrguiNG , 下载后上传并设置
    2. 需要重启才生效
    3. 可以很方便的重命名文件并且添加标签

  4. 添加trackerlist

    1. github搜索 animeTrackerList
    2. 在webui界面的设置-interface-Default tracker list 添加, 格式为raw

autobangumi_tr

  1. dockerhub 搜索 : jqtmviyu/auto_bangumi_tr
  2. 安装上面的教程进行安装
  3. 打开 autobangumi 的web界面看是否连接成功
  4. 添加mikan的rss订阅链接

mikan国内ip访问的地址是: mikanime.tv, 国外访问的是: mikanani.me

但就算是国内访问mikanime.tv也会经常访问不了, 可以用cloudflare worker加速,或者访问别人提供的代理站

例如: https://mikanani.hacgn.fun

emby/jellyfin

  • 如果对硬件解码有需求就选jellyfin
  • 没有的话选emby就可以了, ui功能比较新

具体参考: Emby

避免被吸血

除了迅雷 影音先锋之类的, 还有很多跑pcdn的垃圾人在吸血

所以就算是 qbee, 安装反吸血还是有必要的

github 搜索 PBH-BTN/PeerBanHelper项目, 按照文档设置就可以了.
